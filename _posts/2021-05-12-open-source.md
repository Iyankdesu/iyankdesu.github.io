## Open-Source
-----
Software open-source atau lebih dikenal dengan sebutan FLOSS (Free Libre Open-Source Software) adalah sebutan untuk aplikasi dengan kode sumber yang bersifat terbuka, yaitu source code dari aplikasi tersebut dapat diakses oleh semua orang dari semua kalangan, semua orang memiliki hak untuk mengubah dan melakukan kustomisasi pada source code dari aplikasi tersebut. Berbeda dengan aplikasi dengan sumber yang tidak terbuka, dimana pengguna tidak dapat mengubah isi dari source codenya, pada aplikasi open-source bebas untuk melakukannya.

Umumnya aplikasi - aplikasi open-source itu gratis, tapi ada beberapa aplikasi yang mengharuskan pengguna untuk membayar agar aplikasi tersebut dapat digunakan. Open-source bukan berarti gratis, karena tujuan dari aplikasi open-source adalah membuat program yang sumbernya dapat dilihat serta diubah oleh orang lain.

Contoh software open-source yang mengharuskan pengguna untuk membayar aplikasi adalah sebuah sistem operasi Red Hat Enterprise Linux (RHEL), dimana pengguna harus membayar/berlangganan untuk mendapatkan dukungan dari RHEL.

## Kelebihan Open-Source

-----

Berikut adalah beberapa kelebihan dari software open-source:
1. Kode sumbernya dapat diakses oleh semua orang
2. Umumnya gratis, jadi tidak perlu membayar untuk menggunakannya
3. Sulit untuk dibajak
4. Dapat didistribusikan ulang
5. Lebih ringan

## Sejarah Open-Source Software
-----
Sejarah open-source software pertama kali dikemukaan oleh Richard Stallman, seorang pria kelahiran 19 maret 1953 yang merupakan pendiri dari GNU Project yang merupakan proyek open-source software. 

## Lisensi Open-Source
-----

Terdapat sangat banyak lisensi open-source, tetapi lisensi yang paling sering digunakan adalah lisensi GNU GPL, MIT License, Apache License dan MPL.

1. GNU GPL
   
   GNU GPL (GNU General Public License) adalah lisensi yang mengusung kebebasan untuk mengubah dan membagikan sumber kode dari suatu perangkat lunak. GNU GPL merupakan lisensi yang termasuk dalam copyleft, yaitu kebalikan dari copyright. Lisensi ini didesain untuk membuat kebebasan dalam mendistribusikan salinan dari suatu perangkat lunak serta mengubahnya menjadi suatu perangkat lunak baru, ketika mengubah dan mendistribusikan sebuah perangkat lunak dengan lisensi GNU GPL maka perangkat lunak yang baru diubah pun harus menggunakan lisensi GNU GPL untuk menjaga setiap orang tetap memiliki kebebasan dalam mengakses sumber kodenya. 
   
   <a href="https://www.gnu.org/licenses/gpl-3.0.html" class="button special icon fa-search">Read More</a>

2. MIT License
   
   MIT License adalah lisensi perangkat lunak bebas yang mengizinkan untuk membuat perangkat lunak menjadi berpemilik selama memberitahukan hak cipta dari lisensi MIT. Lisensi MIT kompatibel dengan lisensi copyleft seperti GNU GPL, sehingga perangkat lunak dengan lisensi MIT dapat di integrasikan dengan lisensi GNU GPL, namun tidak bisa sebaliknya.

   <a href="https://opensource.org/licenses/MIT" class="button special icon fa-search">Read More</a>

3. Apache License
   
   Apache License adalah lisensi perangkat lunak yang memungkinkan orang untuk melakukan apapun terhadap perangkat lunaknya, seperti mengubah sumber kode, mendistribusikan ulang, dan lainnya. Lisensi ini tidak termasuk dalam copyleft, sehingga orang yang mendistribusikan perangkat lunak dengan lisensi Apache tidak harus membuka sumber kodenya.

   <a href="https://www.apache.org/licenses/LICENSE-2.0" class="button special icon fa-search">Read More</a>

4. MPL

   MPL (Mozilla Public License) adalah lisensi copyleft yang memperbolehkan untuk memodifikasi sumber kode dari suatu perangkat lunak tanpa harus mendistribusikan ulang perangkat lunak tersebut, berbeda GNU GPL yang mengharuskan untuk mendistribusikan sumber kode yang telah dimodifikasi.

   <a href="https://www.mozilla.org/en-US/MPL/" class="button special icon fa-search">Read More</a>

   Masih ada banyak lisensi open-source yang bisa digunakan, yang termasuk copyleft maupun yang non-copyleft.

## Contoh Software Open-Source

| Closed-Source      | Open-Source   |
| ------------------ | ------------- |
| Adobe Photoshop    | GIMP          |
| Adobe Illustrator  | Inkscape      |
| Adobe Premiere Pro | Kdenlive      |
| Adobe Audition     | Audacity      |
| Microsoft Office   | Libre Office  |
| Google Chrome      | Brave Browser |
| Windows            | Linux         |

Beberapa software di atas hanyalah contoh dari sekian banyaknya software open-source yang lain.

Menggunakan open-source software bukan berarti harus meninggalkan closed-source software, karena ada beberapa hal yang sulit dilakukan karena belum terbiasa dengan open-source software. Pilihan akhirnya ada ditangan pengguna ingin menggunakan open-source atau closed-source, tapi ada baiknya mencoba menggunakan open-source software dari pada closed-source software bajakan.

Sangat banyak pengguna software berbayar yang lebih memilih untuk menggunakan software bajakan dari pada harus membeli software tersebut, salah satu kasus yang paling banyak ditemui adalah pembajakan Windows, Microsoft Office, ataupun software - software dari Adobe. Dari pada menggunakan software bajakan yang ada kemungkinan memiliki virus didalamnya, lebih baik mencoba menggunakan open-source software.
